Headless browser for
AI agents

A fast, persistent CLI for navigating, reading, and interacting with web pages — designed to be called by LLMs one command at a time.

$ brew install juanheyns/gstack-browse/browse
macOS · Linux · Windows ~100ms per command Playwright · Chromium MIT license

Installation

Homebrew (recommended)

brew tap juanheyns/gstack-browse
brew install browse
browse setup-chromium   # installs Chromium once

Manual

# Download a release binary from GitHub and put it in your PATH
curl -L https://github.com/juanheyns/gstack-browse/releases/latest/download/browse-darwin-arm64.tar.gz | tar xz
mv browse /usr/local/bin/
browse setup-chromium

Optional: Claude sidebar agent

brew install juanheyns/gstack-browse/browse-agent
# Enables a Claude-powered sidebar in headed mode. Requires the claude CLI.

Sidebar agent

browse-agent adds a Claude-powered chat panel to headed mode. Type a message and Claude reads the page, runs browse commands, and responds — all without leaving the browser.

Setup

# 1. Install the add-on (requires claude CLI at claude.ai/download)
brew install juanheyns/gstack-browse/browse-agent

# 2. Launch headed Chrome with sidebar from your project directory
cd ~/my-project
browse connect

# 3. Open the sidebar — click the extension icon or press Cmd+Shift+Y

How it works

You type in sidebar
      ↓
Chrome extension → browse server (/sidebar-command)
      ↓
sidebar-agent polls queue → spawns: claude -p "your message"
      ↓
Claude streams response → server → extension → sidebar

Claude runs in your project directory and has access to all browse commands — it can navigate, snapshot, click, and read whatever page is open in Chrome. Each browser tab gets its own independent Claude session.

Quick start

Navigate and read

browse goto https://example.com
browse text                   # extract all visible text
browse snapshot               # structured snapshot with @ref labels
browse screenshot /tmp/page.png

Fill a form

browse goto https://example.com/login
browse fill "#email" "user@example.com"
browse fill "#password" "secret"
browse click "[type=submit]"
browse wait --networkidle
browse url                    # confirm redirect

Use snapshot refs

browse snapshot               # labels interactive elements @e1, @e2 …
browse click @e3
browse fill @e4 "search term"
browse snapshot -D            # diff against previous snapshot

Responsive screenshots

browse goto https://example.com
browse responsive /tmp/shots  # saves mobile / tablet / desktop PNGs
